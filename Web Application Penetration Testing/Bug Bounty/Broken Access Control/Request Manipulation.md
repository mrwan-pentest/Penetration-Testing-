## Request Manipulation is the process of intercepting and modifying an HTTP request before it reaches the server.

Instead of sending the original request, an attacker changes parameters, cookies, headers, or other request data to test how the application responds.

---

# Common Examples

- Changing URL parameters
    
- Modifying POST data
    
- Editing cookies
    
- Changing HTTP headers
    

---

# Why Is It Important?

## Request Manipulation helps identify vulnerabilities such as:

- Broken Access Control
    
- IDOR
    
- Price Manipulation
    
- Authentication Bypass
    
- Business Logic Flaws
    

---

# Common Tool

## Burp Suite is the most popular tool for intercepting and modifying HTTP requests.
