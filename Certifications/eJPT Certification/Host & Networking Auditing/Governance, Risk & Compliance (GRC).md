

هذا الموضوع يتكلم عن شيء مهم جدًا بالشركات اسمه:

```
GRC
```

واختصار:

- Governance
- Risk
- Compliance

وهو يعتبر:

```
إطار لإدارة الأمن داخل الشركة
```

يعني كيف الشركة:

- تدير الأمن
- تتعامل مع المخاطر
- تلتزم بالقوانين والمعايير

---

# أول شيء: Governance 

معناها:

```
الحوكمة أو إدارة الأمن
```

يعني:  
كيف الشركة تنظم الأمن داخليًا.

---

# يشمل أشياء مثل:

- إنشاء سياسات أمنية
- تحديد الصلاحيات
- من المسؤول عن ماذا
- وضع قواعد وإجراءات

---

# مثال 

الشركة تقول:

- يجب استخدام MFA(Multi-Factor Auth)
- كلمات المرور لازم قوية
- لا أحد يدخل السيرفر إلا بتصريح

هذه تعتبر:

```
Governance
```

---

# ثاني شيء: Risk 

معناه:

```
إدارة المخاطر
```

يعني:  
الشركة تحاول تعرف:

- ما هي التهديدات؟
- ما هي الثغرات؟
- ما تأثيرها؟

---

# ثم تحاول تقلل الخطر 

مثلاً:  
وجدوا:

```
RDP مفتوح للإنترنت
```

فيقولون:

- هذا خطر
- قد يؤدي لاختراق

ثم يطبقوا:

- VPN
- Firewall
- MFA

وهذا يسمى:

```
Risk Mitigation
```

أي:  
تقليل الخطر.

---

# ثالث شيء: Compliance 

معناه:

```
الالتزام بالقوانين والمعايير
```

مثل:

- PCI DSS
- GDPR
- HIPAA

---

# يعني الشركة لازم:

- تطبق شروط معينة
- تحمي البيانات
- تتبع قوانين الأمن

---

# مثال 

شركة بطاقات دفع لازم:

```
تلتزم بـ PCI DSS
```

إذا لم تلتزم:

- قد تُغرّم
- أو تُمنع من العمل

---

# الآن لماذا GRC مهم للـ Pentester؟ 

لأن الـ Pentester ليس فقط:

```
يبحث عن ثغرات
```

بل يجب يفهم:

- سياسات الشركة
- المخاطر
- القوانين
- الأولويات

---

# مثال 
وجدت:

```
SQL Injection
```

إذا الشركة تتعامل مع:

- بطاقات بنكية
- بيانات حساسة

فالخطورة تصبح:

```
أعلى بكثير
```

بسبب:

- Compliance
- Risk

---

# أيضًا يساعد بالتقارير 

بدل أن تقول فقط:

```
يوجد SQL Injection
```

تقول:

```
الثغرة قد تسبب مخالفة PCI DSS وتسريب بيانات العملاء
```

وهذا احترافي أكثر.

---

# الخلاصة 

## Governance

كيف الشركة تدير الأمن والسياسات.

---

## Risk

كيف تكتشف وتقلل المخاطر.

---

## Compliance

كيف تلتزم بالقوانين والمعايير.

---

# وGRC كامل يساعد الشركة على:

```
إدارة الأمن بشكل منظم واحترافي
```
