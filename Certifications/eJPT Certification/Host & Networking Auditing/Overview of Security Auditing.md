
## ما هو Security Auditing؟

هو:

```
فحص وتقييم أمن الأنظمة والشبكات
```

لاكتشاف:

- الثغرات
- الإعدادات الخاطئة
- نقاط الضعف

---

# الهدف الأساسي 

```
معرفة مدى أمان النظام
```

قبل أن يستغله مهاجم.

---

# ماذا يشمل؟ 

- فحص الشبكة
- مراجعة الخدمات
- اكتشاف الثغرات
- مراجعة الصلاحيات
- التحقق من التحديثات
- مراجعة الإعدادات الأمنية

---

# الفرق بين Audit و Pentest 

|Audit|Pentest|
|---|---|
|تقييم أمني|اختراق فعلي|
|اكتشاف مشاكل|استغلال مشاكل|
|Defensive|Offensive|

---

# أشياء مهمة ب

## Vulnerability

ثغرة أو ضعف أمني.

---

## Misconfiguration

إعداد أمني خاطئ.

---

## Risk

احتمالية وتأثير الاستغلال.

---

## False Positive

نتيجة تقول توجد ثغرة لكنها غير موجودة.

يعني:

```
الأداة تقول توجد ثغرة… لكن بالحقيقة لا توجد
```


---

## False Negative

يعني:

```
الثغرة موجودة لكن الأداة لم تكتشفها
```

---

# الهدف النهائي 

```
تحسين أمان الأنظمة وتقليل المخاطر
```
