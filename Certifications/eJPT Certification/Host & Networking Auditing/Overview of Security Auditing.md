# Security Auditing

## What is Security Auditing?

Security Auditing is:

```
The process of examining and evaluating the security of systems and networks
```

Its purpose is to identify:

- Vulnerabilities
- Security misconfigurations
- Weaknesses in the environment

Security Auditing helps organizations understand their current security posture and identify areas that need improvement.

---

# Main Objective of Security Auditing

The primary goal is:

```
Determining how secure a system is
```

before attackers can discover and exploit security weaknesses.

---

# What Does Security Auditing Include?

Security Auditing may include:

- Network assessment
- Service review
- Vulnerability discovery
- Permission and access review
- Update and patch verification
- Security configuration review

These activities help identify weaknesses and ensure that security controls are properly implemented.

---

# Difference Between Audit and Pentest

|Audit|Pentest|
|---|---|
|Security evaluation|Real-world exploitation|
|Identifies security issues|Exploits security issues|
|Defensive approach|Offensive approach|

An Audit focuses on evaluating security controls and compliance, while a Pentest focuses on simulating real attacks to determine how vulnerabilities can be exploited.

---

# Security Auditing Phases

## 1. Information Gathering

The first phase focuses on collecting information about the target environment.

This may include:

- Network information
- System details
- Available services
- Infrastructure information

---

## 2. Enumeration

Enumeration involves actively gathering detailed information about discovered systems and services.

Examples:

- Open ports
- Running services
- User accounts
- System information

---

## 3. Vulnerability Assessment

This phase focuses on identifying security weaknesses.

Examples:

- Missing patches
- Vulnerable services
- Weak configurations
- Known vulnerabilities

---

## 4. Configuration Review

This phase examines security settings and configurations.

Examples:

- Firewall rules
- User permissions
- Access controls
- Security policies

---

## 5. Reporting

The final phase documents the audit results.

The report usually includes:

- Identified vulnerabilities
- Security risks
- Evidence
- Recommendations for improvement

# Important Security Terms

## Vulnerability

A Vulnerability is:

```
A security weakness or flaw that can be exploited
```

Examples:

- Outdated software
- Weak passwords
- Vulnerable services

---

## Misconfiguration

A Misconfiguration is:

```
An incorrect or insecure system configuration
```

Examples:

- Default credentials
- Incorrect firewall rules
- Excessive user permissions

---

## Risk

Risk represents:

```
The likelihood and impact of a security issue being exploited
```

Risk evaluation helps organizations prioritize which issues should be fixed first.

---

## False Positive

A False Positive occurs when:

```
A tool reports that a vulnerability exists, but the vulnerability is not actually present
```

Example:

A vulnerability scanner identifies a vulnerable service, but manual verification shows that the system is secure.

---

## False Negative

A False Negative occurs when:

```
A vulnerability exists, but the tool fails to detect it
```

This is dangerous because organizations may believe their systems are secure while a real vulnerability remains.

---

# Final Goal of Security Auditing

The ultimate objective of Security Auditing is:

```
Improving system security and reducing security risks
```

By identifying weaknesses early, organizations can fix security issues before they are exploited by attackers.
