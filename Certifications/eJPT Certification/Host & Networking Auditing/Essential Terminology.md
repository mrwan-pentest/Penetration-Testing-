## 1) Security Policies

```
سياسات أمنية تحدد قواعد وإجراءات الحماية داخل الشركة
```

مثل:

- سياسة كلمات المرور
- صلاحيات المستخدمين
- استخدام الأجهزة

---

# 2) Compliance

```
الالتزام بالمعايير والقوانين الأمنية
```

مثل:

- ISO 27001
- PCI-DSS
- GDPR

---

# 3) Vulnerability

```
ثغرة أو نقطة ضعف يمكن استغلالها
```

مثل:

- SMBv1
- SQL Injection
- كلمات مرور ضعيفة

---

# 4) Control

```
إجراء أمني للحماية وتقليل المخاطر
```

مثل:

- Firewall
- Antivirus
- MFA
- Access Control

---

# 5) Risk Assessment

```
تقييم وتحليل المخاطر الأمنية
```

يعني:

- ما هي التهديدات؟
- ما تأثيرها؟
- ما احتمال حدوثها؟

ثم تحديد:

```
ما الأولوية في الحماية
```

## 6) Audit Trail

```
سجل زمني للأحداث والأنشطة داخل النظام
```

مثل:

- تسجيل الدخول
- تغيير الملفات
- تشغيل الأوامر

يفيد في:

```
تتبع ما حدث ومن قام به
```

---

# 7) Compliance Audit

```
فحص للتأكد من التزام الشركة بالمعايير والسياسات الأمنية
```

مثل:

- ISO
- PCI-DSS
- GDPR

---

# 8) Access Control

```
التحكم بمن يستطيع الوصول وماذا يمكنه أن يفعل
```

مثل:

- Users
- Permissions
- Roles
- MFA

---

# 9) Audit Report

```
تقرير نهائي يحتوي نتائج التدقيق الأمني
```

ويشمل:

- الثغرات
- المخاطر
- التوصيات
- التحسينات المطلوبة
