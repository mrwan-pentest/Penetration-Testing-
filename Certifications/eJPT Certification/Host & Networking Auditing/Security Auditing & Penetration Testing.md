
---

# أولًا: ما هو Security Audit؟ 

الـ Security Audit فكرته:

```
تقييم أمان الشركة بشكل عام
```

يعني يفحص:

- السياسات
- الصلاحيات
- الإعدادات
- الالتزام بالمعايير
- الحماية

ويرى:

```
هل الشركة ماشية بشكل آمن أو لا
```

---

# أما الـ Penetration Test 🔥

فكرته مختلفة.

هو:

```
محاكاة هجوم حقيقي
```

المختبر يحاول:

- استغلال الثغرات
- الدخول للنظام
- تجاوز الحماية

كأنه مهاجم حقيقي.

# الفرق الأساسي بينهم 

## الـ Audit يسأل:

```
هل الشركة تطبق الحماية بشكل صحيح؟
```

---

## أما الـ Pentest يسأل:

```
هل أستطيع اختراق النظام فعليًا؟
```

---

# الـ Audit يكون شامل أكثر 

يفحص:

- السياسات
- القوانين
- الشبكة
- الصلاحيات
- الإجراءات

---

# أما الـ Pentest 

يركز أكثر على:

- الثغرات التقنية
- الاستغلال
- الهجوم
---
# Sequential Approach 

هذا يعني:

```
Audit ثم Pentest
```

وهذا الأكثر شيوعًا.

---

# Combined Approach 

يعني:

```
دمج الـ Audit والـ Pentest معًا
```

في تقييم واحد.
