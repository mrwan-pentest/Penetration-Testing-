

هي أداة تقوم بـ:

```
نسخ (Mirror) موقع كامل إلى جهازك
```

يعني تحمل:

- صفحات HTML
- الصور
- CSS
- JavaScript
- ملفات الموقع

حتى تتصفح الموقع:

```
بدون إنترنت
```

---

# الفكرة الأساسية

بدل ما تزور الموقع كل مرة:

```
HTTrack يحمل نسخة كاملة منه
```

ويعيد بناء الروابط داخليًا.

---

# لماذا تستخدم في الـ Pentesting؟ 

لأن أحيانًا الموقع يحتوي:

- ملفات مخفية
- تعليقات HTML
- نسخ احتياطية
- بيانات داخل صفحات غير واضحة

وعند عمل Mirror:

```
تظهر ملفات مثيرة للاهتمام
```

---

# مثال

لو الموقع يحتوي:

```
backup.zip
config.bak
hidden comments
```

قد تحملها الأداة تلقائيًا.

---

# كيف تعمل؟

الأداة:

1. تزور الصفحة الرئيسية
2. تجمع الروابط
3. تدخل كل رابط
4. تحمل الملفات
5. تعيد العملية تلقائيًا

---

# مثال استخدام

```
httrack http://example.com
```

---

# ماذا يحدث بعد التشغيل؟

ينشئ مجلد يحتوي:

- HTML files
- Images
- CSS
- JS
- ملفات أخرى

---

# ملفات مهمة تنتجها الأداة

|الملف|الوظيفة|
|---|---|
|index.html|الصفحة الرئيسية|
|hts-log.txt|سجل العمليات|
|hts-cache|الكاش|
|cookies.txt|الكوكيز|

# أشياء مهمة 

## Mirror

يعني:

```
نسخة مطابقة للموقع
```

---

## Crawling

يعني:

```
التنقل بين الروابط تلقائيًا
```

---

# كيف تبحث عن أشياء مهمة بعد التحميل؟

---

## عرض الملفات

```
find .
```

---

## البحث عن Flags

```
grep -R "FL@G" .
```

---

## البحث عن تعليقات HTML

```
grep -R "<!--" .
```

---

## البحث عن كلمات حساسة

```
grep -Ri "password" .
```

---

# أشياء قد تكشفها HTTrack

|الشيء|لماذا مهم|
|---|---|
|ملفات backup|قد تحتوي كلمات مرور|
