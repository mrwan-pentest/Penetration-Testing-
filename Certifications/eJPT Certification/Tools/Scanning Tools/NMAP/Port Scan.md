
# `-sS` 

يعني:

```
TCP SYN Scan
```

ويُعرف أيضًا باسم:

```
Half-Open Scan
```


# كيف يعمل؟

Nmap يرسل:

```
SYN
```

إذا المنفذ مفتوح:  
يرد الهدف بـ:

```
SYN-ACK
```

ثم Nmap:

```
لا يكمل الاتصال
```

ويرسل:

```
RST
```

- سريع
- مشهور جدًا
- أقل إزعاجًا في الـ Logs أحيانًا

# مثال

```
nmap -sS 192.168.1.10
```

# `-sT`

يعني:

```
TCP Connect Scan
```


يكمل:

```
TCP 3-Way Handshake كامل
```

يعني:

```
SYN → SYN-ACK → ACK
```

# ثم يغلق الاتصال.

# مثال

```
nmap -sT 192.168.1.10
```

# متى يستخدم؟

عندما:

```
لا تملك صلاحيات Root
```

أوضح لل logs


# `-sU`

UDP Scan.

يفحص:

```
UDP Ports
```

مثل:

- DNS → 53
- SNMP → 161

```
nmap -sU target
```

# `-v`

Verbose Mode.

يعطي تفاصيل أكثر أثناء الفحص.

```
nmap -v target
```


## `-F`

فحص سريع:  
يفحص أشهر 100 بورت فقط بدل 1000.


# `-p-`

يفحص:

```
جميع البورتات
```

من:

```
1-65535
```

بدل أشهر 1000 بورت.

```
nmap -p- target
```

# `-sL` 

يعني:

```
List Scan
```

# ماذا يفعل؟

Nmap:

```
يعرض الأهداف فقط
```

بدون:

- Port Scan
- Ping
- إرسال Packets حقيقية للفحص


# مثال

```
nmap -sL 192.168.1.0/24
```


Nmap يقوم:

- بحساب كل الـ IPs داخل الشبكة
- وعرضها لك فقط
# الناتج مثلًا

```
Nmap scan report for 192.168.1.1Nmap scan report for 192.168.1.2Nmap scan report for 192.168.1.3
```
