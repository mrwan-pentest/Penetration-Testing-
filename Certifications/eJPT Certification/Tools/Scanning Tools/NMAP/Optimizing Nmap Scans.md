
# `--host-timeout` 

يستخدم لـ:

```
تحديد أقصى وقت لفحص Host واحد
```



أحيانًا جهاز معين:

- بطيء جدًا
- لا يرد
- يسبب تأخير للفحص كامل

فيأتي هذا الخيار ويقول لـ Nmap:

```
"إذا تجاوز هذا الوقت، اترك الجهاز وانتقل لغيره"
```


# مثال

```
nmap --host-timeout 30s 192.168.1.10
```
# يمكن استخدام:

| القيمة | المعنى  |
| ------ | ------- |
| `s`    | seconds |
| `m`    | minutes |
| `h`    | hours   |

# `--scan-delay`


يستخدم لـ:

```
إضافة تأخير بين كل Packet والأخرى
```

بدل أن يرسل Nmap الـ Packets بسرعة،  
يقوم بالانتظار قليلًا بين كل إرسال.

حتى:
- يقلل الضجيج
- يتجنب IDS/IPS
- يمنع Rate Limiting
- يجعل الفحص أهدأ (Stealthier)
# مثال

```
nmap --scan-delay 1s 192.168.1.10
```

# ماذا يعني؟

انتظر:

```
1 ثانية
```

بين كل Packet.
# يمكن استخدام:

|القيمة|المعنى|
|---|---|
|`ms`|milliseconds|
|`s`|seconds|
