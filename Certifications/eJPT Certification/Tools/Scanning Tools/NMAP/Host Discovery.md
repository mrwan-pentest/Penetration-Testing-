
اكتشاف الأجهزة الحية بالشبكة


-sn
```
Host Discovery Only
```

يعني:

```
اكتشاف الأجهزة الحية فقط
```

بدون:

- Port Scan
- Service Detection
---


-PS

```
TCP SYN Ping
```



# ماذا يفعل؟

بدل ICMP Ping،  
Nmap يرسل:

```
SYN Packet
```

مثل بداية TCP Handshake.


بعض الأجهزة:

- تمنع ICMP
- لكن تسمح TCP

# مثال

```
nmap -PS 192.168.1.0/24
```

# تحديد Port معين

```
nmap -PS80,443 192.168.1.0/24
```



# ماذا يعني؟

يرسل SYN إلى:

- Port 80
- Port 443


# مفيد جدًا

لأن:

```
TCP Ping يتجاوز أحيانًا الـ Firewalls
```
---

--send-ip

```
يجبر Nmap على استخدام IP Packets مباشرة
```

بدل:

```
Ethernet Frames
```



# الوضع الطبيعي

داخل LAN،  
Nmap غالبًا يستخدم:

```
ARP / Ethernet
```



# مع --send-ip

يستخدم:

```
Raw IP
```

# مثال

```
nmap --send-ip -sn 192.168.1.0/24
```
---

# `-PA` 

هذا الخيار يعني:

```
TCP ACK Ping
```


# ماذا يفعل؟

بدل إرسال:

```
SYN
```

مثل `-PS`

فهو يرسل:

```
ACK Packet
```


# ما هي ACK؟

ACK اختصار:

```
Acknowledgment
```

ومعناها:

```
تأكيد الاستلام
```



# لكن لماذا نرسل ACK والجهاز لم يفتح اتصال أصلًا؟ 
هذه هي الفكرة الذكية.

إذا الجهاز حي،  
فسيرد غالبًا بـ:

```
RST
```

وهذا يخبر Nmap أن:

```
الجهاز موجود ويعمل
```


# مثال

```
nmap -PA 192.168.1.0/24
```

# تحديد Port معين

```
nmap -PA80,443 192.168.1.0/24
```


# لماذا نستخدم ACK Ping؟

لأن بعض الـ Firewalls:

- تمنع ICMP
- تمنع SYN
- لكن تسمح ACK

---
# `-PE`


يعني:

```
ICMP Echo Ping
```


# ماذا يفعل؟

Nmap يرسل:

```
ICMP Echo Request
```

مثل أمر:

```
ping
```


# إذا رد الجهاز بـ:

```
ICMP Echo Reply
```

فهذا يعني:

```
Host is Up
```


# مثال

```
nmap -PE 192.168.1.0/24
```


# لماذا يستخدم؟

لاكتشاف:

```
الأجهزة الحية
```


# مهم جدًا

بعض الأجهزة أو الـ Firewalls:

```
تمنع ICMP
```

---
# `-Pn`

يخبر Nmap:

```
لا تعمل Ping Discovery
```

يعني اعتبر الهدف:

```
Alive
```

حتى لو ما يرد على ICMP.

مفيد إذا:

- الجدار الناري يمنع Ping
- الهدف لا يرد على ICMP

```
nmap -Pn target
```
