
# لماذا نحتاج HTTPS؟

المشكلة في HTTP العادي:

```
HTTP يرسل البيانات كنص عادي (Clear Text)
```

مثال:

```
POST /login HTTP/1.1
username=admin
password=123456
```

إذا كان هناك مهاجم في الشبكة:

```
Wireshark
tcpdump
MITM
```

فيمكنه رؤية البيانات مباشرة.

---

# ما هو HTTPS؟

HTTPS اختصار:

```
HyperText Transfer Protocol Secure
```

وهو:

```
نسخة آمنة من HTTP
```

تستخدم لتأمين الاتصال بين:

```
المتصفح ↔ السيرفر
```

---

# كيف يعمل؟

في HTTP العادي:

```
Browser   
 │    
  HTTP   
  │
  Web Server
```

---

في HTTPS:

```
Browser    │    │ HTTP    │ SSL/TLS    │Web Server
```

أي أن HTTP أصبح يعمل داخل طبقة حماية إضافية.

---

# ما هو SSL/TLS؟

## SSL

اختصار:

```
Secure Sockets Layer
```

تقنية قديمة.

---

## TLS

اختصار:

```
Transport Layer Security
```

النسخة الحديثة والأكثر استخداماً.


عملياً عندما نقول:

```
SSL Certificate
```

غالباً نقصد:

```
TLS Certificate
```

---

# ماذا تضيف TLS؟

تعطي HTTP ثلاثة أشياء مهمة جداً:


## 1. Confidentiality

### السرية

تعني:

```
لا أحد يستطيع قراءة البيانات أثناء انتقالها
```

مثال:

قبل التشفير:

```
password=123456
```

بعد التشفير:

```
a8f91x0d8f1s0df81...
```

المهاجم يرى بيانات مشفرة فقط.

---

## 2. Integrity

### سلامة البيانات

تعني:

```
لا يمكن تعديل البيانات أثناء النقل
```

مثال:

أرسلت:

```
amount=100
```

المهاجم يحاول تغييرها إلى:

```
amount=10000
```

TLS يكتشف التعديل.

---

## 3. Authentication

### التحقق من الهوية

تعني:

```
التأكد أنك متصل بالسيرفر الحقيقي
```

وليس:

```
سيرفر مزيف
```

لهذا يستخدم الموقع شهادة (Certificate).
