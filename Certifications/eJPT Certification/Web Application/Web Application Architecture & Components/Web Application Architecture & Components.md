
##  Web Technologies ?

كمختبر اختراق ستتعامل يوميًا مع:

- HTML
- JavaScript
- Cookies
- APIs
- Databases
- Web Servers

لذلك يجب أن تعرف:

- أين تعمل كل تقنية؟
- ما وظيفتها؟
- كيف يمكن استغلالها؟

---

# Client-Side Technologies

هي التقنيات التي تعمل داخل متصفح المستخدم.


## HTML

لغة بناء صفحات الويب.

تستخدم لإنشاء:

- العناوين
- الأزرار
- النماذج
- الجداول

مهم للمختبر لأنك قد تجد:

- Hidden Fields
- Comments
- معلومات حساسة داخل الصفحة

---

## CSS

مسؤول عن شكل الصفحة.

يحدد:

- الألوان
- الخطوط
- الأحجام
- ترتيب العناصر

نادراً ما يحتوي ثغرات مباشرة لكنه قد يكشف ملفات أو مسارات.

---

## JavaScript

لغة البرمجة التي تعمل داخل المتصفح.

تستخدم لـ:

- التفاعل مع المستخدم
- التحقق من المدخلات
- تحديث الصفحة دون إعادة تحميلها

مهمة جدًا لأن ملفات JS قد تحتوي:

- API Endpoints
- مفاتيح API
- وظائف مخفية
- منطق يمكن تجاوزه

---

## Cookies

ملفات صغيرة يخزنها المتصفح.

غالباً تستخدم لـ:

- Sessions
- تذكر المستخدم
- الإعدادات

مهمة لأن سرقتها قد تؤدي إلى:

- Session Hijacking

---

## Local Storage

مكان لتخزين البيانات داخل المتصفح.

الفرق عن Cookie:

- لا ترسل تلقائياً للسيرفر.
- تبقى داخل المتصفح.

قد تحتوي:

- JWT Tokens
- API Keys

---

# Server-Side Technologies

هي التقنيات التي تعمل على السيرفر.

---

## Web Server

أمثلة:

- Apache
- Nginx
- IIS

وظيفته:

- استقبال HTTP Requests
- إرسال الملفات الثابتة
- تمرير الطلبات للتطبيق

فكر فيه كـ "الباب الأمامي للموقع".

---

## Application Server

يشغل التطبيق نفسه.

وظيفته:

- تنفيذ الكود
- معالجة الطلبات
- التواصل مع قاعدة البيانات

فكر فيه كـ "عقل التطبيق".

---

## Database Server

مكان تخزين البيانات.

أمثلة:

- MySQL
- PostgreSQL
- MSSQL
- Oracle

يخزن:

- المستخدمين
- كلمات المرور
- المنتجات
- الطلبات

أشهر ثغرة مرتبطة به:

```
SQL Injection
```

---

## Server-Side Languages

لغات تشغيل التطبيق.

أشهرها:

- PHP
- Python
- Java
- Ruby

معرفة اللغة تساعدك على فهم:

- الملفات
- الأخطاء
- الثغرات المحتملة

---

# Communication & Data Flow

آلية التواصل:

```
Browser        
↓
HTTP Request   
↓
Server        
↓
Database      
↓
HTTP Response  
↓
Browser
```



---

# Data Interchange

يقصد به تبادل البيانات بين الأنظمة المختلفة.

مثال:

- موقع يتواصل مع PayPal.
- تطبيق يستخدم Google Maps.

كل هذا يعتمد على تبادل البيانات.

---

# API

اختصار:

```
Application Programming Interface
```

واجهة تسمح للأنظمة بالتواصل.

مثال:

تطبيق الطقس يطلب البيانات من API خاص بالأرصاد.

كمختبر اختراق تعتبر APIs هدفاً أساسياً للفحص.

---

# Data Formats

## JSON

أكثر صيغة استخداماً حالياً.

مميزاته:

- بسيط
- سريع
- سهل القراءة

يستخدم بكثرة مع APIs.

---

## XML

صيغة أقدم وأكثر تفصيلاً.

تستخدم في:

- SOAP
- بعض الأنظمة القديمة

مرتبطة بثغرات مثل:

```
XXE
```

---

# REST

أشهر أسلوب لبناء APIs.

يعتمد على HTTP Methods.

---

## GET

جلب البيانات.

---

## POST

إرسال بيانات جديدة.

---

## PUT

تعديل البيانات.

---

## DELETE

حذف البيانات.


---

## SOAP

بروتوكول أقدم يعتمد على XML.

أقل انتشاراً من REST لكنه ما زال موجوداً في بعض الشركات والأنظمة القديمة.

---

# Security Technologies

## Authentication

التحقق من الهوية.

السؤال:

```
من أنت؟
```

مثال:

- Username
- Password

---

## Authorization

التحقق من الصلاحيات.

السؤال:

```
ماذا يسمح لك أن تفعل؟
```

مثال:

هل المستخدم العادي يستطيع دخول لوحة الإدارة؟

---

## SSL/TLS

المسؤول عن تشفير الاتصال.

يحوّل:

```
HTTP
```

إلى:

```
HTTPS
```

يحمي:

- كلمات المرور
- Sessions
- البيانات الحساسة

أثناء انتقالها عبر الشبكة.

---

# External Technologies

## CDN

اختصار:

```
Content Delivery Network
```

شبكة سيرفرات موزعة حول العالم.

وظيفتها:

- تسريع تحميل الموقع.
- تقليل الضغط على السيرفر الأساسي.

---

## Third-Party Libraries

مكتبات جاهزة يستخدمها المطورون.

أمثلة:

- jQuery
- Bootstrap
- React Packages

إذا كانت قديمة قد تحتوي على ثغرات معروفة.
