
# أولًا الفكرة العامة 

في شبكات ويندوز:

- الأجهزة تحتاج تشارك:
    - ملفات
    - طابعات
    - فولدرات
    - users

وهنا يأتي:

- SMB
- NetBIOS

---

# أولًا: ما هو SMB؟

SMB اختصار:

```
Server Message Block
```

وهو:

> بروتوكول مشاركة الملفات والطابعات في ويندوز.

---

# تخيله كذا 

لما تفتح:

```
\\192.168.1.10\share
```

أنت تستخدم SMB.


# ماذا يسمح لك؟

- قراءة ملفات
- رفع ملفات
- طباعة
- Remote IPC
- الوصول لـ shares


# البورتات المهمة

|البورت|الاستخدام|
|---|---|
|445|SMB الحديث|
|139|SMB عبر NetBIOS|

---

# الآن ما هو NetBIOS؟ 👀

NetBIOS أقدم من SMB.

وهو ليس file sharing نفسه.

بل:

> نظام يساعد الأجهزة تتعرف على بعضها داخل الشبكة.


# يعني وظيفته:

- أسماء الأجهزة
- جلسات الاتصال
- broadcasting
- discovery


# مثال 👀

بدل ما تعرف IP الجهاز:

```
192.168.1.20
```

تعرفه باسم:

```
WORKSTATION-PC
```

هذا جزء من NetBIOS.


# خدمات NetBIOS

|الخدمة|البورت|
|---|---|
|Name Service|137|
|Datagram|138|
|Session|139|

# العلاقة بين SMB و NetBIOS 

قديمًا:

```
SMB كان يعمل فوق NetBIOS
```

يعني:

- NetBIOS يساعد بالاتصال
- SMB ينقل الملفات


# الآن؟

الأنظمة الحديثة تستخدم:

```
SMB مباشرة عبر 445
```

بدون الحاجة لـ NetBIOS غالبًا.

لكن:

- NetBIOS ما زال موجود للتوافق مع الأنظمة القديمة.
