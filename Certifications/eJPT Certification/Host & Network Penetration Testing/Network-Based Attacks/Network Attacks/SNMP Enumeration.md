
# ما هو SNMP؟

اختصار:

```
Simple Network Management Protocol
```

يعني:

> بروتوكول لإدارة ومراقبة أجهزة الشبكة.

# مين يستخدمه؟

- Routers
- Switches
- Printers
- Servers
- Firewalls


# ليش موجود؟

حتى الأدمن يقدر:

- يعرف حالة الجهاز
- يراقب الشبكة
- يجمع معلومات
- يعرف إذا الجهاز فيه مشكلة


# المشكلة الأمنية 

كثير أجهزة:

- تترك الإعدادات الافتراضية
- تستخدم Community Strings ضعيفة

مثل:

```
publicprivate
```


# ما هي Community String؟

هي تقريبًا:

> كلمة مرور SNMP

# أشهر الأنواع

|القيمة|الاستخدام|
|---|---|
|public|قراءة فقط|
|private|قراءة وكتابة|


# المنافذ 

|المنفذ|الاستخدام|
|---|---|
|UDP 161|Queries|
|UDP 162|Traps|

# ما معنى Trap؟

يعني:

> الجهاز يرسل تنبيه تلقائي.

مثلاً:

- CPU مرتفع
- Interface down


# مكونات SNMP


## 1- SNMP Manager

هو:

> الجهاز الذي يسأل.

مثلاً:

- Kali
- Monitoring Server


## 2- SNMP Agent

هو:

> البرنامج الموجود على الراوتر أو السيرفر ويرد على الطلبات.

---

## 3- MIB

اختصار:

```
Management Information Base
```

وهو:

> قاعدة بيانات تحتوي معلومات الجهاز.


# ما هو الـ OID؟

كل معلومة لها رقم مميز.

مثل:

```
1.3.6.1.2.1.1.5
```

هذا قد يمثل:

> اسم الجهاز.


# أداة snmpwalk

أشهر أداة.

`snmpwalk` أداة تُستخدم للاستعلام من أجهزة فيها SNMP مفعّل، وسحب معلومات كثيرة دفعة واحدة من الـ MIB.

بمعنى:  
بدل ما تسأل الجهاز سؤال واحد فقط،  
تقوم الأداة “تمشي” داخل قاعدة معلومات SNMP وتجمع البيانات.

لهذا اسمها:

> walk = تمشي/تتجول


```
snmpwalk -v 1 -c public 192.168.1.10
```

شرحها:

|الجزء|المعنى|
|---|---|
|snmpwalk|الأداة|
|-v 1|إصدار SNMP|
|-c public|community string|
|IP|الهدف|

```
snmpwalk -v1 -c public 192.168.1.10 1.3.6.1.2.1.1.5
```


# معلومات النظام

```
snmpwalk -v1 -c public 192.168.1.10 1.3.6.1.2.1.1.1
```


# المستخدمين (أحياناً)

```
snmpwalk -v1 -c public 192.168.1.10 1.3.6.1.4.1.77.1.2.25
```
